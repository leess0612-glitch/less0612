# 안티그라비티 - 업무 자동화 툴 기획서

**작성일**: 2026-02-24
**대상**: 소규모 팀 (2~10명)
**구성**: 총 4개 독립 모듈, 단일 웹앱으로 통합 제공

---

## 전체 아키텍처 방향

### 통합 방식
4개 도구를 별도 앱이 아닌 **하나의 웹앱(대시보드 형태)** 으로 구성.
좌측 사이드바 네비게이션으로 각 모듈 전환.

### 기술 스택 (공통)
| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | React + TypeScript | 안정성, 생태계 |
| 빌드 | Vite | 빠른 개발 속도 |
| 스타일 | Tailwind CSS | 빠른 UI 구성 |
| 상태관리 | Zustand | 가볍고 직관적 |
| 데이터 저장 | IndexedDB (Dexie.js) | 로컬, 서버 불필요 |
| 차트 | Recharts | React 친화적 |
| 엑셀 처리 | SheetJS (xlsx) | 읽기/쓰기 모두 지원 |
| 이미지 편집 | Fabric.js | Canvas 기반, 레이어 지원 |

### 배포
- 로컬 실행 (npm run dev) 또는 Electron으로 데스크탑 앱화 가능
- 서버/DB 없이 완전 로컬 동작

## 1. 인터넷 상품정리 및 마진 계산기

### 개요
네이버 카페 기반 온라인 판매 상품의 원가, 판매가, 각종 비용을 관리하고
실시간으로 순이익 및 마진율을 계산하는 도구.

### 핵심 문제 정의
- 상품이 많아질수록 원가/마진 파악이 어려움
- 배송비, 결제수수료 등 숨은 비용 누락으로 실제 마진 착각
- 상품별 수익성 비교가 어려워 우선순위 판단 불가

### 주요 기능

#### 1-1. 상품 등록/관리
- 상품명, 카테고리, 상품 URL(카페 링크), 상태(판매중/품절/중단)
- 상품 이미지 썸네일 등록
- 등록일, 최종 수정일 자동 기록
- 메모/특이사항 필드

#### 1-2. 비용 구조 입력
| 항목 | 설명 |
|---|---|
| 매입가 | 공급처 구매 원가 |
| 매입 배송비 | 공급처 → 나에게 오는 배송비 |
| 포장재비 | 박스, 완충재 등 |
| 발송 배송비 | 나 → 고객 배송비 (착불/선불 설정) |
| 네이버페이 수수료 | 결제 수수료 % 입력 |
| 카페 광고비 | 상품별 할당 광고비 (선택) |
| 기타비용 | 자유 항목 추가 가능 |

#### 1-3. 마진 계산 엔진
```
판매가 - (매입가 + 매입배송비 + 포장재비 + 발송배송비 + 결제수수료 + 기타) = 순이익
순이익 / 판매가 × 100 = 마진율(%)
```
- 판매가 변경 시 실시간 재계산
- 손익분기 판매가 역산 기능 (목표 마진율 입력 → 필요 판매가 출력)

#### 1-4. 상품 목록 대시보드
- 테이블 뷰: 상품명, 판매가, 원가, 순이익, 마진율 한눈에 보기
- 마진율 색상 표시 (빨강: 10% 미만 / 노랑: 10~20% / 초록: 20% 이상)
- 정렬: 마진율 높은 순, 판매가 순, 등록일 순
- 필터: 카테고리, 상태, 마진율 범위

#### 1-5. 통계 패널
- 전체 상품 수 / 판매중 상품 수
- 평균 마진율
- 마진율 상위 5개 상품
- 마진율 하위 5개 상품 (적자 위험 경고)

#### 1-6. 데이터 관리
- 엑셀 내보내기 (전체 상품 + 계산 결과)
- 엑셀 가져오기 (템플릿 제공)
- 상품 복사 기능 (유사 상품 빠르게 등록)

### 화면 구성
```
[상단] 검색바 | 카테고리 필터 | 상태 필터 | + 상품 추가 버튼
[좌측] 통계 패널 (마진율 분포 차트)
[우측] 상품 테이블
[하단] 엑셀 내보내기 버튼
```

## 2. 이미지요소 합성기

### 개요
블로그 업로드용 상품 이미지를 빠르게 제작할 수 있는 캔버스 기반 편집 도구.
포토샵 없이 브라우저에서 바로 합성, 저장.

### 핵심 문제 정의
- 블로그마다 이미지 규격/스타일이 다름
- 매번 포토샵 열기 번거로움
- 같은 템플릿을 여러 상품에 반복 적용하는 작업이 많음

### 주요 기능

#### 2-1. 캔버스 기본 설정
- 사이즈 프리셋: 블로그 표준(860×N), 정사각형(1:1), 세로형(3:4) 등
- 사용자 정의 사이즈 입력
- 배경색 / 배경 이미지 설정

#### 2-2. 요소 추가
| 요소 | 세부 기능 |
|---|---|
| 이미지 | 로컬 파일 업로드, 투명도 조절, 크롭, 반전 |
| 텍스트 | 폰트 선택, 크기, 색상, 굵기, 그림자, 외곽선 |
| 도형 | 사각형, 원, 삼각형, 색상/테두리 설정 |
| 스티커 | 기본 제공 아이콘 세트 (화살표, 별, 뱃지 등) |
| 워터마크 | 반투명 텍스트/로고 삽입 |

#### 2-3. 레이어 관리
- 레이어 목록 패널 (포토샵과 유사)
- 레이어 순서 드래그로 변경
- 레이어 잠금 / 숨김 / 삭제
- 레이어 복사

#### 2-4. 편집 도구
- 드래그로 위치 이동
- 모서리 핸들로 크기 조절
- 회전 핸들
- 정렬: 좌/우/상/하/가운데 정렬, 균등 분배
- 실행 취소 / 재실행 (Ctrl+Z / Ctrl+Y)

#### 2-5. 템플릿 시스템
- 현재 작업물을 템플릿으로 저장
- 템플릿 불러와서 이미지만 교체
- 기본 제공 템플릿: 상품 소개형, 가격 강조형, 비교형

#### 2-6. 배치 처리
- 이미지 여러 장 업로드
- 같은 템플릿(텍스트, 워터마크 등)을 모든 이미지에 자동 적용
- 한번에 전체 저장

#### 2-7. 내보내기
- JPG (품질 조절 슬라이더)
- PNG (투명 배경 지원)
- 블로그 최적화 사이즈 자동 압축 옵션

### 화면 구성
```
[상단] 파일 | 편집 | 뷰 | 내보내기
[좌측] 요소 추가 패널 (이미지/텍스트/도형/스티커)
[중앙] 캔버스 편집 영역
[우측] 레이어 패널 + 선택 요소 속성 패널
```

## 3. 콜 통계 사이트

### 개요
고객 문의 인바운드 콜 데이터를 엑셀/CSV로 업로드하여
팀 전체의 콜 응대 현황을 시각화하고 분석하는 대시보드.

### 핵심 문제 정의
- 콜 기록이 엑셀에만 있어 트렌드 파악이 어려움
- 팀원별 응대량, 처리율 비교가 수동 집계로 번거로움
- 시간대별 피크 타임 파악 불가로 인력 배치 비효율

### 주요 기능

#### 3-1. 데이터 업로드
- 엑셀(.xlsx, .xls) / CSV 파일 업로드
- 컬럼 매핑 UI: 파일의 어떤 컬럼이 "날짜", "담당자", "통화시간" 인지 사용자가 지정
- 업로드 이력 관리 (여러 파일 누적 가능)
- 중복 데이터 감지 및 경고

#### 3-2. 지원 데이터 항목
| 항목 | 설명 |
|---|---|
| 날짜/시간 | 콜 발생 일시 |
| 담당자 | 응대한 팀원 이름 |
| 통화시간 | 초 또는 분 단위 |
| 처리결과 | 해결/미해결/콜백/부재 등 |
| 문의유형 | 배송/반품/상품문의 등 (선택) |
| 고객연락처 | 선택 (통계용, 개인정보 마스킹) |

#### 3-3. 메인 대시보드
**요약 카드**
- 오늘 총 콜 수
- 평균 통화시간
- 해결률 (%)
- 미처리 건수

**차트**
- 일별 콜 수 추이 (라인 차트)
- 시간대별 콜 분포 히트맵 (0~24시)
- 담당자별 응대 건수 (바 차트)
- 처리결과 비율 (도넛 차트)
- 문의유형 분포 (파이 차트)

#### 3-4. 담당자별 상세
- 담당자 선택 시 개인 통계 드릴다운
- 처리 건수, 평균 통화시간, 해결률
- 월별 추이

#### 3-5. 기간 필터
- 오늘 / 이번 주 / 이번 달 / 지난 달
- 사용자 정의 날짜 범위 선택

#### 3-6. 리포트 내보내기
- 현재 화면 PDF 저장
- 요약 데이터 엑셀 내보내기
- 차트 이미지 저장 (PNG)

### 화면 구성
```
[상단] 기간 필터 | 담당자 필터 | 데이터 업로드 버튼
[상단] 요약 카드 4개 (총 콜/평균시간/해결률/미처리)
[중간] 일별 추이 차트 (와이드)
[하단 좌] 시간대별 히트맵
[하단 중] 담당자별 바차트
[하단 우] 처리결과 도넛차트
```

## 4. 렌탈 상품정리 및 마진 계산기

### 개요
렌탈(대여) 상품의 재고를 관리하고,
렌탈 요금 대비 원가 회수 기간 및 수익성을 분석하는 도구.

### 핵심 문제 정의
- 렌탈은 단순 판매와 달리 감가상각, 수리비 등 장기 비용이 발생
- 얼마나 대여해야 본전인지 파악하기 어려움
- 재고 상태(대여중/반납/수리) 추적이 수동으로 복잡

### 주요 기능

#### 4-1. 상품 등록
- 상품명, 카테고리, 모델명, 일련번호
- 구매가, 구매일
- 사진 등록

#### 4-2. 비용 구조 입력
| 항목 | 설명 |
|---|---|
| 구매가 | 초기 투자 비용 |
| 예상 내용연수 | 몇 년/몇 회 사용 가능 |
| 월 유지보수비 | 정기 점검, 소모품 비용 |
| 보험료 | 월 보험료 (선택) |
| 수리비 누계 | 발생 시마다 입력 |

#### 4-3. 렌탈 요금 설정
- 일 단위 요금
- 주 단위 요금 (할인율 적용)
- 월 단위 요금 (할인율 적용)
- 보증금 설정 (선택)

#### 4-4. 수익성 계산 엔진
```
총 투자비용 = 구매가 + 누적 수리비 + (월 유지보수비 × 운영 개월)
총 렌탈 수익 = 렌탈 건별 수익 합산
순수익 = 총 렌탈 수익 - 총 투자비용
손익분기점 = 총 투자비용 ÷ 일 요금 → 필요 대여일 수
```
- 현재까지 대여일 기준 회수율 (%)
- 예상 완전 회수까지 남은 기간

#### 4-5. 대여 기록 관리
- 대여 등록: 고객명, 연락처, 대여 시작일, 반납 예정일, 요금
- 반납 처리: 실제 반납일, 상태 점검 결과
- 연체 계산: 반납 예정일 초과 시 자동 표시
- 대여 이력 전체 조회

#### 4-6. 재고 현황 대시보드
- 전체 상품 수
- 상태별 현황: 대여가능 / 대여중 / 수리중 / 폐기
- 오늘 반납 예정 건 알림
- 연체 건 목록 강조 표시

#### 4-7. 수익 분석
- 상품별 누적 수익 / 회수율 비교
- 월별 렌탈 수익 추이
- 가장 수익성 높은 상품 TOP 5
- 회수율 낮은 상품 (투자 대비 저조) 경고

### 화면 구성
```
[상단] 재고 현황 카드 (가능/대여중/수리중/연체)
[중간 좌] 상품 목록 테이블 (상태, 회수율 표시)
[중간 우] 선택 상품 상세 (비용구조 + 수익성 계산)
[하단] 대여 기록 + 수익 추이 차트
```

## 공통 UX 원칙

1. **오프라인 우선**: 인터넷 없어도 모든 기능 동작
2. **데이터 영구 보존**: 브라우저 닫아도 데이터 유지 (IndexedDB)
3. **엑셀 연동**: 모든 모듈 엑셀 내보내기/가져오기 지원
4. **반응형 불필요**: PC 전용 최적화 (팀 내부 도구)
5. **빠른 입력**: 키보드 단축키, 자동완성 우선 지원
