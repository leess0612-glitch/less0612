# 안티그라비티 - 공통 아키텍처 및 개발 방향

**작성일**: 2026-02-24
**대상**: 소규모 팀 (2~10명)
**구성**: 총 4개 독립 모듈, 단일 웹앱으로 통합 제공

---

## 전체 아키텍처 방향

### 통합 방식
4개 도구를 별도 앱이 아닌 **하나의 웹앱(대시보드 형태)** 으로 구성.
좌측 사이드바 네비게이션으로 각 모듈 전환.

### 기술 스택 (공통)
| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | React + TypeScript | 안정성, 생태계 |
| 빌드 | Vite | 빠른 개발 속도 |
| 스타일 | Tailwind CSS | 빠른 UI 구성 |
| 상태관리 | Zustand | 가볍고 직관적 |
| 데이터 저장 | IndexedDB (Dexie.js) | 로컬, 서버 불필요 |
| 차트 | Recharts | React 친화적 |
| 엑셀 처리 | SheetJS (xlsx) | 읽기/쓰기 모두 지원 |
| 이미지 편집 | Fabric.js | Canvas 기반, 레이어 지원 |

### 배포
- 로컬 실행 (npm run dev) 또는 Electron으로 데스크탑 앱화 가능
- 서버/DB 없이 완전 로컬 동작

---

## 개발 순서 (권장)

| 순서 | 모듈 | 이유 |
|---|---|---|
| 1순위 | 인터넷 마진 계산기 | 가장 빠른 실무 적용 가능 |
| 2순위 | 렌탈 마진 계산기 | 구조 유사, 빠르게 개발 가능 |
| 3순위 | 콜 통계 | 데이터 구조 설계 필요 |
| 4순위 | 이미지 합성기 | 기술 난이도 가장 높음 |

---

## 공통 UX 원칙

1. **오프라인 우선**: 인터넷 없어도 모든 기능 동작
2. **데이터 영구 보존**: 브라우저 닫아도 데이터 유지 (IndexedDB)
3. **엑셀 연동**: 모든 모듈 엑셀 내보내기/가져오기 지원
4. **반응형 불필요**: PC 전용 최적화 (팀 내부 도구)
5. **빠른 입력**: 키보드 단축키, 자동완성 우선 지원
