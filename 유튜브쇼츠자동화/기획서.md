# 휴대폰 IT/밈 쇼츠 채널 자동화 기획서

작성일: 2026-06-08

## 1. 배경 및 목표
- 참고 영상: https://www.youtube.com/watch?v=arrKfg0V268
  ("이제 클로드 코드가 기획자이자 편집자입니다..." / 채널: 빌더 조슈 Builder Josh)
  - ※ 최초에는 자막/스크립트 추출이 불가능했으나(유튜브 JS 렌더링), 사용자가 직접 스크립트를
    확보·전달해주어 실제 워크플로우를 분석 완료함 → **7. 참고 영상 실제 워크플로우 분석** 참고.
    이 사람은 정보성 롱폼(경제/과학/심리학/역사 등) 채널을 Claude Code 기반 멀티에이전트로
    완전 자동화 운영 중이며, 본 기획(휴대폰 IT 쇼츠)에 그대로 적용 가능한 실전 노하우가 다수 확인됨.
- 목표: 휴대폰 관련 IT 뉴스/이슈/밈을 쇼츠 영상으로 제작하는 채널을 운영하며
  1) 광고/제휴 수익화
  2) 본인 사업체 "핸드폰성지카페" 홍보
  를 동시에 달성. 전 과정을 자동화하여 본업에 지장 없이 운영.

## 2. 사용자 제약 조건 (★ 가장 중요)
- 영상 제작/편집 경험: **전혀 없음 (완전 처음)**
- 투입 가능 시간: **거의 0에 가까움** (본업이 바빠서 물리적으로 시간 투입 불가)
  - → "직접 몇 개 만들어보며 감 잡기(Phase 0 수동 제작)"는 불가능. 처음부터 자동화 중심으로 설계해야 함.
- 예산: 월 5~20만원 (~$35-150/month)
- 툴/스택: 추천받고 싶음 (현재 보유 도구 없음)

## 3. 콘텐츠 전략
- 포맷: 30~50초 쇼츠
- 업로드 주기: 처음엔 주 3~4회 → 점차 매일 업로드로 확장
- 홍보 비율: 핸드폰성지카페 홍보 영상은 전체의 약 1/5~1/7 빈도로 삽입 (시청자 이탈 방지)
- 차별화: 단순 자동생성 티가 나면 스팸성으로 분류되어 수익화 심사에서 불리할 수 있음
  → "검증된 패턴" 기반 기획으로 퀄리티 확보 필요

### 3-1. 콘텐츠 유형별 구조 설계 (★ 메인 콘텐츠 vs 홍보 콘텐츠 분리)
두 콘텐츠는 성격이 달라 같은 틀로 찍어내면 안 되고, 각자에 맞는 구조를 따로 적용한다.

**[메인 콘텐츠] IT 이슈/정보/밈 — 유연한 가이드라인**
- 소재마다 설명 분량과 임팩트 포인트가 달라서 초 단위로 고정하면 어색해짐
- → Phase 0'(벤치마킹 분석)에서는 "초 단위 템플릿"이 아니라
  **구성 요소와 순서에 대한 유연한 가이드라인**(예: 후킹 → 핵심 정보 → 마무리 임팩트, 단 분량은 소재 따라 가변)을 추출

**[홍보 콘텐츠] 단가표 기반 카페 유입 영상 — 정형화된 템플릿**
- 핸드폰성지카페가 보유한 "매일 갱신되는 단가표"는 데이터 포맷이 항상 일정함
  → 초 단위로 고정한 템플릿을 그대로 써도 어색하지 않고, 오히려 자동화하기 쉬움
- 구조 (Hook-Body-Bridge-CTA, 30초 기준):
  | 구간 | 시간 | 내용 |
  | :--- | :--- | :--- |
  | Hook | 00:00~00:03 | IT 이슈 자극적 요약으로 시선 끌기 (예: "아이폰 케이스 단종 진짜 이유") |
  | Body | 00:03~00:20 | 벤치마킹한 정보성 영상 구조를 복제한 유용한 정보 전달 |
  | Bridge | 00:20~00:25 | "근데 이 기종, 오늘 우리 동네 성지에서는 얼마일까?" → 화면 전환, 당일 최저가 단가표 그래픽 노출 |
  | CTA | 00:25~00:30 | "프로필 링크 누르고 카페 가입하면 오늘 단가표 즉시 확인" (혜택 교환형 CTA로 고정) |
- 핵심 아이디어: 비전 LLM으로 매일 수십 개 업체의 비정형 단가표 이미지에서 가격 데이터를 자동 추출 →
  "오늘의 최저가"를 콘텐츠화. 이는 (a) 아무나 따라할 수 없는 사용자만의 고유 데이터이고,
  (b) 시청자에게 실질적으로 유용하며, (c) "정보 → 가격 → 가입"으로 자연스럽게 이어지는
  전환 동선을 만들어, 기존의 "1/5~1/7 비율로 별도 홍보 영상 끼워넣기"보다 훨씬 자연스럽고
  전환율도 높을 것으로 기대됨.
- (출처: 제미나이 기획안 "인스타그램 릴스 기반 핸드폰 성지 카페 유입 자동화 시스템 구축 기획서 v2"의
  핵심 구조를 본 채널의 "홍보 영상 슬롯" 설계로 채택)

## 4. 로드맵 (수정본 — Phase 0 수동 제작 단계 제거)

### Phase 0' — 시장 리서치 자동화 (사람 시간 ≈ 0)
- 사람이 직접 만들어보는 대신, Claude가 휴대폰 IT/밈 쇼츠 카테고리에서
  이미 잘 되고 있는 채널·영상을 분석하여 패턴을 추출
  - 자주 쓰이는 후킹 멘트, 영상 길이/구성/자막 스타일, 조회수 잘 나오는 주제 유형 등
- 메인 콘텐츠(이슈/정보/밈)는 소재마다 편차가 커서 **"유연한 가이드라인"**으로,
  홍보 콘텐츠(단가표 기반)는 데이터 포맷이 일정해서 **"초 단위 고정 템플릿"**(3-1 참고)으로
  — 두 가지 산출물을 분리해서 정리
- 효과: 시행착오 없이 처음부터 검증된 패턴 기반으로 시작 가능

### Phase 1 — 소재수집 + 대본 자동화
- 메인 콘텐츠: 트렌드/이슈 소재를 자동 수집하고, Phase 0'의 유연한 가이드라인에 맞춰 대본 자동 생성
- 홍보 콘텐츠: 비전 LLM으로 당일 단가표 데이터를 추출 → Hook-Body-Bridge-CTA 고정 템플릿에 대입해 대본 자동 생성

### Phase 2 — 음성/이미지 자동 생성
- TTS(타입캐스트/ElevenLabs)로 음성 생성
- AI 이미지(빙 이미지 크리에이터/Leonardo AI)로 비주얼 소스 생성

### Phase 3 — 영상 자동 편집
- Claude Code + ffmpeg로 코드 기반 자동 편집/합성 (자막은 Whisper/캡컷 자동화)

### Phase 4 — 업로드/메타데이터 자동화
- 제목/설명/태그 자동 생성 및 업로드 자동화
- 추후 확장 시 Creatomate 등 활용 검토

### Phase 5 — 분석 피드백 루프
- 조회수/시청 지속시간 등 데이터를 분석해 다음 기획에 반영 (자동화)

## 5. 사람의 개입 방식 (최소화 설계)
- "제작"이 아니라 "가끔 결과물 확인 후 한 줄 피드백"으로 한정
  - 예: 주 1회, 그 주 영상 1~2개만 보고 "톤 괜찮다 / 별로다" 정도의 짧은 코멘트
  - Claude가 그 피드백을 다음 기획에 반영하여 시스템이 스스로 개선
- 본업 중 5분 내외로 충분하도록 설계

## 6. 다음 단계 (대기 중)
- Phase 0' 착수: Claude Code로 유사 채널/영상 분석 → 대본 템플릿 및 기획 가이드라인 생성
- 사용자 확인 후 진행 예정

## 7. 참고 영상 실제 워크플로우 분석 (★ 사용자가 스크립트 직접 확보 → 실전 검증된 사례)

영상 속 운영자(전직 알고리즘 강사)는 정보성 롱폼 채널을 Claude Code 멀티에이전트로 완전
자동화하여 운영 중. 아래 구조는 본 기획에 그대로 이식 가능한 "실전 검증된 설계도"임.

### 7-1. 오케스트레이션 구조 — "유튜브 PD" 메인 에이전트
- 메인 스킬(유튜브 PD)이 전체 워크플로우를 관장하되, **세부 작업은 직접 하지 않고 다른
  에이전트에게 위임**. 대본/이미지/영상/편집/TTS를 각각의 전문 에이전트가 협업해서 완성.
- **컨텍스트 전달 방식**: 단계별 결과물을 **MD 파일로 저장** → 다음 에이전트 실행 시 그 MD를
  넘겨주고, 결과도 다시 MD로 받는 방식. (워크플로우를 "정해진 형태"로 명확히 줘서 스킬이
  순서대로 체크하며 진행하게 하는 패턴 — "AI가 그때그때 판단" 방식과 대비됨)
- → 본 기획의 Phase별 산출물도 MD 파일로 표준화해서 다음 단계 에이전트에 전달하는 구조를
  채택할 것 (예: Phase 0' 결과물 = `리서치_가이드라인.md`, Phase 1 결과물 = `대본_초안.md` 등)

### 7-2. 제작 공정 전체 흐름
1. **레퍼런스 수집**: 같은 주제의 타 영상 중 조회수가 높은 것들의 URL을 수집
   → 파이썬으로 대본·썸네일·제목·댓글까지 자동 수집
2. **분석 → 패턴 파악**: 에이전트들이 "왜 이 영상이 성공했는가" 분석 → 패턴 도출
   (단, 영상 속 정보에 허위가 섞여 있을 수 있어 **웹검색으로 팩트체크 필수**)
3. **전략 설계**: 전략가 에이전트가 타겟층별 컨셉 후보를 추천 → 운영자가 선택/조합
4. **훅·인트로 설계**: 클릭률에 가장 큰 영향 → 대본 작성 전에 먼저 설계
5. **대본 작성**: 기획서 → 초안 → 검수 → 최종 스크립트 (대본에만 에이전트 약 6개 분업 투입)
6. **TTS 음성 + 자막 생성**: ElevenLabs API (음성+자막 파일 동시 수령)
7. **씬 설계(스토리보드)**: 대본+기획서를 근거로 자막을 일정 간격으로 분할해 장면 구성
8. **이미지/영상 프롬프트 생성 → 생성**: 장면별 이미지/영상 자동 생성
9. **JSON 기반 캡컷 자동 편집**: 편집 정보를 JSON으로 명세 → 캡컷이 프로젝트로 자동 인식
10. **업로드 자동화**: 유튜브 공식 API로 썸네일/제목/설명 등 자동 업로드

### 7-3. 핵심 인사이트 (★ 본 기획에 직접 적용 가능)

**① "대본이 9할, 이미지 퀄리티는 의외로 안 중요"**
- AI 이미지로 만든 롱폼 영상을 보면, 화면과 음성이 안 맞아도 조회수 10만~30만이 나오는 경우가 많음
  → 시청자는 생각보다 이미지 퀄리티/싱크에 둔감, **대본·썸네일·음성이 핵심 변수**
- → 본 기획도 한정된 리소스(예산·관심)를 대본/썸네일/음성 품질에 집중 배분하고,
  이미지는 "가성비 라인"으로 선택하는 게 합리적 (실제로 운영자도 최고급 대신 가성비 조합 사용)

**② 대본 제작의 "분업화" 이유 — 한 세션이 아니라 여러 에이전트로 쪼갠다**
- 대본을 한 세션·한 에이전트가 처음부터 끝까지 쓰면 글자수도 못 채우고 뒤로 갈수록 퀄리티가 떨어짐
- "기획서"만 탄탄하게 만들어두면, 그걸 기준으로 각 파트(훅, 본문, 마무리 등)를 별도
  에이전트가 따로 써도 일관성이 유지됨 → **Phase 1의 "대본 자동화"를 단일 작업이 아니라
  [기획서 작성 → 파트별 초안 → 검수 → 최종 통합]의 분업 파이프라인으로 설계할 것**

**③ 전략·선택까지 AI에게 위임하는 게 더 나은 결과를 낳았다**
- 운영자도 처음엔 "전략은 사람이 세워야지"라고 생각했으나, AI가 추천한 3개 후보 중 AI 자신의
  1순위 추천이 항상 무난했고, 반대로 **본인이 "이게 잘 될 것 같다"고 베팅한 건 90%가 틀림**
- 결국 전략가 에이전트에게 선택권까지 위임하고 본인은 손을 뗌 ("오토 모드")
- → 본 기획의 "5. 사람의 개입 방식" 설계와 정확히 같은 방향. 단, 한발 더 나아가
  **"피드백을 주는 것"조차 부담스러우면, 전략가 에이전트의 1순위 추천을 그대로 채택하는
  완전 오토 모드도 충분히 작동할 수 있음** — 시간이 0에 가까운 본 사용자 상황에 특히 유효한 옵션

**④ JSON 기반 캡컷(CapCut) 자동 편집 — ffmpeg보다 현실적인 대안**
- 캡컷은 프로젝트를 JSON 형식으로 관리한다는 점에 착안 (다른 서비스의 JSON 편집 API에서 아이디어 획득)
- Duration/Start 등 타이밍, 리소스 경로, 확대·전환 등 모션 효과까지 **전부 JSON으로 명세 가능**
- 방법: `Resources` 폴더에 이미지·영상·음성·자막을 넣고, 편집 내용을 명세한 JSON과 함께
  캡컷의 드래프트(프로젝트) 폴더에 통째로 넣으면 → 캡컷이 자동으로 프로젝트로 인식
  → 사람은 캡컷에서 **열기 + Export(렌더링)** 두 클릭만 하면 끝 (유일한 수동 개입 지점)
- → 본 기획의 **Phase 3(영상 자동 편집)을 "Claude Code + ffmpeg" 대신 "Claude Code가
  캡컷용 편집 JSON을 자동 생성"하는 방식으로 변경 검토할 것**. 무료 프로그램(캡컷)을 그대로
  활용할 수 있고, 실전에서 검증된 방식이라 리스크가 낮음

**⑤ 씬(장면) 설계 노하우**
- 정보성 콘텐츠는 한 장면을 6~7초 길이로 설계하는 것이 적정 (카테고리마다 다름:
  예를 들어 시니어 타겟의 잔잔한 야담/썰 채널은 정지 이미지로 10~20분을 끌어도 잘 됨)
- 대본+기획서 내용을 근거로 클로드가 자막을 일정 간격으로 나눠 장면을 자동 구성
- → 휴대폰 IT 쇼츠(30~50초)는 호흡이 더 짧으므로, Phase 0'에서 벤치마킹할 때
  "쇼츠에 적합한 장면 길이"를 별도로 패턴화해야 함 (단순히 6~7초를 그대로 적용 X)

**⑥ 가장 어려웠던 기술적 난제 — 음성·자막 동기화**
- ElevenLabs가 음성과 함께 주는 자막 타이밍 자체가 부정확한 경우가 종종 있음
  (대본을 더 정제해서 보내거나 재요청하는 식으로 대응 중이나 완벽하진 않음)
- 자막을 "어느 단위로 끊을 것인가"(문장 단위 vs 절 단위)도 정답이 없는 어려운 문제
- → 본 기획에서도 TTS+자막 파이프라인 구축 시 **이 부분을 가장 큰 리스크 구간으로 미리
  인지하고, 검수 에이전트를 별도로 두는 등 안전장치를 마련할 것**

**⑦ 도메인 지식 학습이 선행되어야 함 — "사용자가 이해한 만큼 AI도 해낸다"**
- 운영자는 AI에게 맡기기 전에 무료 유튜브 강의·미스터비스트의 노하우 등을 먼저 공부하고,
  본인이 이해/정리한 내용을 AI에게 전달하는 방식으로 시작함 (AI에게 "강의 보고 알아서 해" X)
- → 본 사용자는 시간이 없지만, 이 학습은 "직접 영상을 만들어보는 시간"과는 다름.
  예를 들어 출퇴근 시간에 짧은 유튜브 쇼츠 강의 1~2개만 보고 핵심을 메모해 Claude에게
  넘겨주는 정도로도 충분히 효과를 볼 수 있음 — Phase 0'를 보완하는 "가벼운 사전 학습" 단계로
  반영할 가치가 있음 (단, 의무는 아니고 선택적 보강 옵션으로 둘 것)

### 7-4. 비용 구조 (실측 데이터 — 본 기획 예산 검증에 활용)
영상 한 편(약 20분 분량 기준, 8,000~10,000자 대본)당:
- TTS (ElevenLabs): 약 2,500~3,000원
- Claude 사용료: 약 0~1,000원
- 썸네일 생성 (나노바나나 프로 API, 5개): 약 500~1,000원 미만
- **총합: 약 5,000원/편**

→ 본 기획의 콘텐츠는 30~50초 쇼츠로 분량이 훨씬 짧아(대본 글자수 대폭 감소),
**편당 비용은 이보다 더 낮을 가능성이 큼**. 월 5~20만원 예산이면 하루 여러 편도 충분히
가능한 수준 — 예산 제약에 대한 우려는 낮춰도 될 것으로 판단됨.

### 7-5. 종합 — 본 기획서 대비 보강 포인트 요약
| 항목 | 기존 본 기획서 | 실전 사례 반영 후 보강 방향 |
| :--- | :--- | :--- |
| Phase 0' | "벤치마킹 분석"이라는 개념만 존재 | 레퍼런스 수집→분석→팩트체크의 **검증된 세부 절차**로 구체화 |
| Phase 1 (대본) | "자동 생성"이라고만 명시 | **기획서→파트별 분업→검수→통합**의 분업 파이프라인으로 설계 |
| Phase 2 (이미지) | "AI 이미지 생성" | 퀄리티보다 **가성비/속도 우선** 원칙 명문화 (대본이 더 중요하므로) |
| Phase 3 (편집) | "Claude Code + ffmpeg" | **JSON 기반 캡컷 자동 편집**으로 변경 검토 (무료, 실전 검증, 리스크 낮음) |
| Phase 5 / 사람 개입 | "주 1회 한 줄 피드백" | 한 단계 더 나아가 **전략가 에이전트 1순위 추천을 그대로 채택하는
완전 오토 모드**도 옵션으로 추가 (사람의 직감보다 AI 추천이 더 정확했다는 실증 사례 존재) |
| 리스크 관리 | 명시 안 됨 | **음성·자막 동기화**를 최대 리스크 구간으로 사전 인지, 검수 단계 강화 |
| 예산 | "월 5~20만원" (추정치) | 실측 단가(편당 ~5,000원, 본 기획은 분량이 짧아 더 저렴할 것)로 **검증 완료** |
