# 휴대폰 IT/밈 쇼츠 채널 자동화 기획서

작성일: 2026-06-08

## 1. 배경 및 목표
- 참고 영상: https://www.youtube.com/watch?v=arrKfg0V268
  ("이제 클로드 코드가 기획자이자 편집자입니다..." / 채널: 빌더 조슈 Builder Josh)
  - ※ 해당 영상은 자막/스크립트 추출이 불가능하여(유튜브 JS 렌더링) 제목·채널 정보만 확인됨.
    내부 워크플로우/툴 디테일은 알 수 없음 → 본 기획은 그 컨셉(=Claude Code가 기획+편집을 모두 수행)만 차용.
- 목표: 휴대폰 관련 IT 뉴스/이슈/밈을 쇼츠 영상으로 제작하는 채널을 운영하며
  1) 광고/제휴 수익화
  2) 본인 사업체 "핸드폰성지카페" 홍보
  를 동시에 달성. 전 과정을 자동화하여 본업에 지장 없이 운영.

## 2. 사용자 제약 조건 (★ 가장 중요)
- 영상 제작/편집 경험: **전혀 없음 (완전 처음)**
- 투입 가능 시간: **거의 0에 가까움** (본업이 바빠서 물리적으로 시간 투입 불가)
  - → "직접 몇 개 만들어보며 감 잡기(Phase 0 수동 제작)"는 불가능. 처음부터 자동화 중심으로 설계해야 함.
- 예산: 월 5~20만원 (~$35-150/month)
- 툴/스택: 추천받고 싶음 (현재 보유 도구 없음)

## 3. 콘텐츠 전략
- 포맷: 30~50초 쇼츠
- 업로드 주기: 처음엔 주 3~4회 → 점차 매일 업로드로 확장
- 홍보 비율: 핸드폰성지카페 홍보 영상은 전체의 약 1/5~1/7 빈도로 삽입 (시청자 이탈 방지)
- 차별화: 단순 자동생성 티가 나면 스팸성으로 분류되어 수익화 심사에서 불리할 수 있음
  → "검증된 패턴" 기반 기획으로 퀄리티 확보 필요

### 3-1. 콘텐츠 유형별 구조 설계 (★ 메인 콘텐츠 vs 홍보 콘텐츠 분리)
두 콘텐츠는 성격이 달라 같은 틀로 찍어내면 안 되고, 각자에 맞는 구조를 따로 적용한다.

**[메인 콘텐츠] IT 이슈/정보/밈 — 유연한 가이드라인**
- 소재마다 설명 분량과 임팩트 포인트가 달라서 초 단위로 고정하면 어색해짐
- → Phase 0'(벤치마킹 분석)에서는 "초 단위 템플릿"이 아니라
  **구성 요소와 순서에 대한 유연한 가이드라인**(예: 후킹 → 핵심 정보 → 마무리 임팩트, 단 분량은 소재 따라 가변)을 추출

**[홍보 콘텐츠] 단가표 기반 카페 유입 영상 — 정형화된 템플릿**
- 핸드폰성지카페가 보유한 "매일 갱신되는 단가표"는 데이터 포맷이 항상 일정함
  → 초 단위로 고정한 템플릿을 그대로 써도 어색하지 않고, 오히려 자동화하기 쉬움
- 구조 (Hook-Body-Bridge-CTA, 30초 기준):
  | 구간 | 시간 | 내용 |
  | :--- | :--- | :--- |
  | Hook | 00:00~00:03 | IT 이슈 자극적 요약으로 시선 끌기 (예: "아이폰 케이스 단종 진짜 이유") |
  | Body | 00:03~00:20 | 벤치마킹한 정보성 영상 구조를 복제한 유용한 정보 전달 |
  | Bridge | 00:20~00:25 | "근데 이 기종, 오늘 우리 동네 성지에서는 얼마일까?" → 화면 전환, 당일 최저가 단가표 그래픽 노출 |
  | CTA | 00:25~00:30 | "프로필 링크 누르고 카페 가입하면 오늘 단가표 즉시 확인" (혜택 교환형 CTA로 고정) |
- 핵심 아이디어: 비전 LLM으로 매일 수십 개 업체의 비정형 단가표 이미지에서 가격 데이터를 자동 추출 →
  "오늘의 최저가"를 콘텐츠화. 이는 (a) 아무나 따라할 수 없는 사용자만의 고유 데이터이고,
  (b) 시청자에게 실질적으로 유용하며, (c) "정보 → 가격 → 가입"으로 자연스럽게 이어지는
  전환 동선을 만들어, 기존의 "1/5~1/7 비율로 별도 홍보 영상 끼워넣기"보다 훨씬 자연스럽고
  전환율도 높을 것으로 기대됨.
- (출처: 제미나이 기획안 "인스타그램 릴스 기반 핸드폰 성지 카페 유입 자동화 시스템 구축 기획서 v2"의
  핵심 구조를 본 채널의 "홍보 영상 슬롯" 설계로 채택)

## 4. 로드맵 (수정본 — Phase 0 수동 제작 단계 제거)

### Phase 0' — 시장 리서치 자동화 (사람 시간 ≈ 0)
- 사람이 직접 만들어보는 대신, Claude가 휴대폰 IT/밈 쇼츠 카테고리에서
  이미 잘 되고 있는 채널·영상을 분석하여 패턴을 추출
  - 자주 쓰이는 후킹 멘트, 영상 길이/구성/자막 스타일, 조회수 잘 나오는 주제 유형 등
- 분석 결과를 곧바로 "대본 템플릿 + 기획 가이드라인"으로 변환 → Phase 1의 출발점으로 사용
- 효과: 시행착오 없이 처음부터 검증된 패턴 기반으로 시작 가능

### Phase 1 — 소재수집 + 대본 자동화
- 트렌드/이슈 소재를 자동 수집하고, Phase 0'에서 만든 템플릿에 맞춰 대본 자동 생성

### Phase 2 — 음성/이미지 자동 생성
- TTS(타입캐스트/ElevenLabs)로 음성 생성
- AI 이미지(빙 이미지 크리에이터/Leonardo AI)로 비주얼 소스 생성

### Phase 3 — 영상 자동 편집
- Claude Code + ffmpeg로 코드 기반 자동 편집/합성 (자막은 Whisper/캡컷 자동화)

### Phase 4 — 업로드/메타데이터 자동화
- 제목/설명/태그 자동 생성 및 업로드 자동화
- 추후 확장 시 Creatomate 등 활용 검토

### Phase 5 — 분석 피드백 루프
- 조회수/시청 지속시간 등 데이터를 분석해 다음 기획에 반영 (자동화)

## 5. 사람의 개입 방식 (최소화 설계)
- "제작"이 아니라 "가끔 결과물 확인 후 한 줄 피드백"으로 한정
  - 예: 주 1회, 그 주 영상 1~2개만 보고 "톤 괜찮다 / 별로다" 정도의 짧은 코멘트
  - Claude가 그 피드백을 다음 기획에 반영하여 시스템이 스스로 개선
- 본업 중 5분 내외로 충분하도록 설계

## 6. 다음 단계 (대기 중)

- 사용자 확인 후 진행 예정
