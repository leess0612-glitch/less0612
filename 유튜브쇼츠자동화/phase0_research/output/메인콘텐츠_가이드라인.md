## 1. 가장 먼저 확인된 사실 — "포맷 선택 자체가 조회수를 가른다"

조회수 편차가 30회 ~ 448만회로 10만 배 이상 차이남에도, 가장 잘 된 영상들의 성격이 모두 다름:
- **24초 스킬 챌린지형** (448만회) — [한글타자 애국가 4초컷 1906타 ▶](https://www.youtube.com/watch?v=MGL73aqMfqs)
- **9초 밈 패러디형** (297만회) — [Shot on iPhone meme Philippines ▶](https://www.youtube.com/watch?v=62MdPpFTyCI)
- **60초 비유 정보형** (124만회) — [이제부터 하지 맙시다 ▶](https://www.youtube.com/watch?v=Jpa0HfMYo18)

→ 결론: 특정 포맷 하나만 쓰는 게 아니라 **세 가지 패턴을 돌아가며 섞는 전략**이 유리.
포맷마다 조회수가 잘 나오는 이유가 달라서, 한 가지만 고집하면 잠재적 시청자를 놓치게 됨.

## 2. 포맷별 패턴 분석 (유연한 가이드라인 — 초 단위 고정 아님)

### [패턴 A] 스킬/챌린지형 — "이게 말이 돼?" 반응 유발
- **예시**: [한글타자 애국가 4초컷 1906타 ▶](https://www.youtube.com/watch?v=MGL73aqMfqs) (448만회, 24초)
- 구조: 결과(충격적인 수치/퍼포먼스)를 바로 보여줌 → 설명 없음
- 핵심: 제목 자체가 후킹 ("4초컷", "1906타") — 숫자가 직관적으로 임팩트를 전달
- 길이: 20~30초 (설명 불필요, 보여주는 것만으로 충분)
- 휴대폰 채널 적용 예: "아이폰16 카메라 나이트모드 4초만에 이렇게 나옴", "삼성 갤럭시 폴드 펼치는 속도 000타"

### [패턴 B] 밈 패러디형 — "친숙한 포맷의 초압축 변형"
- **예시**: [Shot on iPhone meme Philippines ▶](https://www.youtube.com/watch?v=62MdPpFTyCI) (297만회, 9초)
- 구조: 이미 유행 중인 밈 템플릿 차용 → 지역화/변형 → 펀치라인 하나로 마무리
- 핵심: 대사 4줄 이하, 상황과 타이밍의 임팩트가 전부. "정보"가 아니라 "한 방의 웃음 포인트"
- 길이: 9~25초 (설명 없이 바로 보여줌)
- 휴대폰 채널 적용 예: 유행하는 드라마/영화 밈 템플릿에 "아이폰 vs 갤럭시" 상황 대입

### [패턴 C] 비유 정보형 — "어려운 걸 쉽게 설명"
- **예시**: [이제부터 하지 맙시다 ▶](https://www.youtube.com/watch?v=Jpa0HfMYo18) (124만회, 60초)
- 후킹: 시간 제약 + 직접 명령형 ("딱 1분만 집중해서 들어봐")
  → 시청자에게 "짧게 끝난다"는 안도감과 "집중하라"는 지시를 동시에 줌
- 전개: 어려운 개념(국제법상 영토분쟁)을 누구나 아는 일상 상황(친구가 내 컵을 뺏으려는 상황)으로
  치환해서 설명 → 정보의 양보다 "이해시키는 비유"가 핵심
- 마무리: 명확한 한 줄 결론 + 행동 지침 ("그냥 알박기 시전이 정답")
- 길이: 50~60초
- 휴대폰 채널 적용 예: "통신사가 공시지원금을 올리는 진짜 이유" → 비유로 설명 → "그러니까 이때 사면 됩니다"

### [패턴 D] 리뷰/비교형 — "소통하는 리뷰어 톤" (초기 운영 시 보조 패턴으로 활용)
- **예시**: [갤럭시z플립4 블루 VS 아이폰13 프로 블루 색상대결 ▶](https://www.youtube.com/watch?v=x-UemVozQOM) (1.8만회, 60초)
- 후킹: "~비교해달라는 댓글이 달렸는데" → 시청자 요청에 답하는 형식으로 소통감 부여
- 전개: 두 대상 직접 비교 + 솔직한 주관 의견
- 조회수가 상대적으로 낮아(1.8만) — 팬덤 형성 이후에 더 효과적인 패턴. 초기엔 A·B·C 우선

## 3. 제작 시 핵심 원칙

1. **길이는 소재에 맞게 가변적으로** — 강제로 맞추지 말 것
   - 스킬/밈형: 9~30초
   - 정보형: 50~60초
   - 기획서의 "30~50초 고정" 방침은 → **소재 성격에 맞는 길이를 먼저 정한 뒤 압축**하는 방향으로 수정 권장

2. **후킹은 3가지 형식 중 하나 선택**
   - 숫자/결과 직접 제시형 ("4초컷", "00만원 차이")
   - 시간 제약 + 명령형 ("딱 1분만", "~만 보면 됨")
   - 시청자 참여 유도형 ("댓글로 요청받은~", "~궁금하신 분 있죠?")

3. **정보형은 반드시 "비유"를 먼저 설계한 후 대본 작성**
   → Phase 1 대본 자동화 프롬프트에 "이 정보를 어떤 익숙한 일상 상황에 빗댈 수 있는가"를
   먼저 설계하도록 포함시킬 것

4. **마무리는 항상 명확한 한 줄 결론으로 끝낼 것**
   — 정보 나열 후 애매하게 끝나면 시청 후 만족감이 낮아짐
