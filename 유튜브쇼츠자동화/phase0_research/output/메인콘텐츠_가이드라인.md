# 메인 콘텐츠(IT 이슈/정보/밈) 기획 가이드라인 — Phase 0' 1차 분석 결과

작성 기준: "아이폰 밈/휴대폰 IT 이슈/스마트폰 꿀팁/갤럭시 아이폰 비교" 키워드로 수집한
쇼츠 12편(조회수 23회~297만회) 중 자막 확보 4편 + 메타데이터 전수 분석

## 1. 가장 먼저 확인된 사실 — "포맷 선택 자체가 조회수를 가른다"
- 같은 "휴대폰 IT/밈" 카테고리 안에서도 조회수 편차가 23회 ~ 297만회로 10만 배 이상 차이남
- 가장 잘 된 영상 2개는 성격이 완전히 다름:
  - **9초짜리 밈 패러디** (297만 회, 대사 4줄)
  - **60초짜리 정보 콘텐츠** (124만 회, 풍부한 설명+비유)
  → 결론: "정보냐 밈이냐"보다 **"그 포맷 안에서 얼마나 임팩트 있게 만들었는가"**가 핵심.
    두 가지 포맷 모두 채널의 메인 축으로 쓸 수 있음 (택1이 아니라 혼합 운용 권장)

## 2. 포맷별 패턴 (유연한 가이드라인 — 초 단위 고정 아님)

### [패턴 A] 정보형 — "비유로 설명하기" (예: 1,246,407회 "이제부터 하지 맙시다")
1. **후킹**: 시간 제약 + 직접 명령형 ("딱 1분만 집중해서 들어봐")
   → 시청자에게 "짧게 끝난다"는 안도감과 "집중하라"는 지시를 동시에 줌
2. **전개**: 어려운 개념(국제법상 영토분쟁)을 **누구나 아는 일상 상황(친구가 내 컵을 뺏으려는 상황)**으로
   치환해서 설명 → 정보 콘텐츠의 핵심은 "정보의 양"이 아니라 "이해시키는 비유"
3. **마무리**: 모호하게 끝내지 않고 **명확한 결론 + 행동 지침**으로 마무리
   ("그냥 알박기 시전이 정답") → 시청자가 "한 가지를 확실히 얻어간다"는 느낌을 줌
- 길이: 60초 (정보 밀도가 높을수록 길어져도 무방 — 단, 비유로 늘어지지 않게 압축)

### [패턴 B] 밈형 — "친숙한 포맷의 초압축 패러디" (예: 2,971,800회 "Shot on iPhone meme")
1. 이미 유행 중인 밈 템플릿(예: "Shot on iPhone" 시리즈)을 그대로 차용해 지역화/변형
2. 대사는 최소 (4줄 이하), **상황과 타이밍의 임팩트**가 전부
3. 길이는 9~30초 수준으로 매우 짧음 — "정보"가 아니라 "한 방의 웃음 포인트"
- → 메인 콘텐츠 중 일부를 이런 **초압축 밈 패러디**로 채우면 제작 공수 대비 효율이 매우 높을 수 있음
  (대본이 짧아 TTS·자막 비용도 거의 안 듦)

### [패턴 C] 리뷰/비교형 — "댓글 요청에 답하는 친근한 톤" (예: 18,080회 갤럭시 vs 아이폰)
1. 후킹을 "시청자 댓글 요청에 답한다"는 형식으로 설계 ("~비교해달라는 댓글이 달렸는데")
   → 일방적 정보 전달이 아니라 "소통하는 채널"이라는 인상을 줌
2. 두 대상을 직접 나란히 비교 + 본인의 주관적 의견을 솔직하게 덧붙임 (리뷰어 톤)
- 길이: 60초. 다만 이 영상은 조회수가 상대적으로 낮아(1.8만) — 이 패턴은 **"팬덤 형성 후"**에
  더 효과적일 수 있음 (초기엔 패턴 A·B가 신규 유입에 더 유리해 보임)

## 3. 메인 콘텐츠 제작 시 적용할 핵심 원칙
1. **"정보의 양"보다 "이해시키는 장치(비유, 비교, 패러디)"에 집중**
   — Phase 1 대본 자동화 시, "이 정보를 어떤 익숙한 상황/대상에 빗댈 수 있는가"를
   먼저 설계하도록 프롬프트에 반드시 포함시킬 것
2. **길이는 소재에 맞게 가변적으로 — 강제로 늘리거나 줄이지 말 것**
   - 펀치라인 하나면 충분한 밈: 9~20초
   - 비유로 설명이 필요한 정보: 50~60초
   → "30~50초 쇼츠"라는 기존 기획서의 고정값은 **"이 범위 안에서 맞추려 애쓰지 말고,
   소재 성격에 맞는 길이를 먼저 정한 뒤 그 안에서 압축"**하는 방향으로 수정 권장
3. **후킹은 "명령형 직접 화법" 또는 "시청자 참여 유도형" 중 하나로 선택**
   - 패턴 A 식 ("~해봐", "~만 보면 됨")
   - 패턴 C 식 ("~비교해달라고 해서 알아봤습니다")
4. **마무리는 항상 "명확한 한 줄 결론"으로 끝낼 것**
   — 애매하게 정보만 나열하고 끝나면 시청 후 만족감이 낮아짐 (패턴 A의 핵심 강점)

## 4. 한계 및 추가 검증 필요 사항
- 이번 1차 수집은 표본이 작음(12편, 자막 확보 4편) — 정확한 패턴 확정을 위해서는
  **표본을 30~50편 수준으로 늘려 재분석**하는 것을 권장
- 키워드를 "휴대폰 IT 이슈"처럼 일반적으로 잡으면 뉴스/광고성 영상이 섞여 들어옴
  → 다음 수집 때는 "휴대폰 밈 더빙", "아이폰 vs 갤럭시 드립" 등 더 구체적인 키워드로 보강 필요
- 자막이 없는 영상(특히 고조회수 밈)은 **영상 자체(이미지/타이밍)에 대한 분석이 별도로 필요**
  → 다음 단계에서는 자막뿐 아니라 영상의 장면 전환/타이밍 패턴도 분석 대상에 포함시킬 것
