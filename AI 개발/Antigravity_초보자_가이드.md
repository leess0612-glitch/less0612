## 1단계: 기초 개념 이해

Antigravity를 이해하기 전에, 먼저 알아야 할 기본 개념들입니다.

### 프로그래밍이란?

컴퓨터에게 일을 시키기 위해 작성하는 **명령어의 집합**입니다.

- 사람의 언어: "불 꺼줘"
- 컴퓨터 언어: `light.turnOff()`

컴퓨터는 사람 말을 못 알아들어서, 컴퓨터가 이해하는 언어(프로그래밍 언어)로 써줘야 합니다.

### IDE란?

**Integrated Development Environment** (통합 개발 환경)

쉽게 말해, **코드 작성을 도와주는 고급 메모장**입니다.

일반 메모장과의 차이:
- 코드에 색깔을 입혀서 보기 쉽게 해줌
- 오타나 실수를 알려줌
- 코드 실행 버튼이 있음
- 파일 관리가 편함

### AI란?

**Artificial Intelligence** (인공지능)

사람처럼 생각하고, 배우고, 답변하는 컴퓨터 기술입니다.

- ChatGPT, Gemini, Claude 등이 대표적인 AI
- 질문하면 사람처럼 대답해줌
- 글도 쓰고, 그림도 그리고, 코드도 짤 수 있음

### AI 코딩 도구란?

AI가 **코드 작성을 도와주는 도구**입니다.

- "로그인 페이지 만들어줘" → AI가 코드를 작성해줌
- 코드에서 버그 찾아달라고 하면 찾아줌
- 모르는 코드 설명해달라고 하면 설명해줌

## 2단계: Antigravity 소개

### Antigravity가 뭔가요?

**Google이 2025년 11월에 출시한 AI 코딩 도구**입니다.

핵심 특징:
- **무료**로 사용 가능
- **Gemini 3** AI가 탑재됨
- Windows, Mac, Linux 모두 지원

### 왜 특별한가요?

다른 AI 코딩 도구와의 차이점:

| 일반 AI 코딩 도구 | Antigravity |
|------------------|-------------|
| AI에게 질문 → 답변 받음 | AI에게 명령 → **직접 작업 수행** |
| 코드 복사/붙여넣기 필요 | **자동으로 파일 생성/수정** |
| 테스트는 내가 직접 | **AI가 테스트까지 수행** |
| 한 번에 하나씩 | **여러 작업 동시 진행 가능** |

### Gemini 3란?

Antigravity에 탑재된 **Google의 최신 AI 모델**입니다.

- Gemini 1: 이해하는 AI
- Gemini 2: 추론하는 AI
- **Gemini 3: 행동하는 AI** ← 직접 코딩하고 실행함

### VS Code와의 관계

Antigravity는 **VS Code를 기반으로 만들어졌습니다**.

- VS Code: Microsoft가 만든 유명한 무료 코드 편집기
- Antigravity: VS Code + Google AI를 합친 것
- 장점: VS Code 쓰던 사람은 익숙한 화면

## 3단계: 화면 구성 익히기

Antigravity를 처음 열면 여러 패널이 보입니다. 각각의 역할을 알아봅시다.

```
┌─────────────────────────────────────────────────────────────┐
│  [파일 탐색기]  │        [편집기 - 코드 작성 공간]           │
│                │                                            │
│  📁 프로젝트    │   1  <!DOCTYPE html>                       │
│  ├─ 📄 index   │   2  <html>                                │
│  ├─ 📄 style   │   3    <head>                              │
│  └─ 📁 src     │   4      <title>Hello</title>              │
│                │   5    </head>                             │
├────────────────┼────────────────────────────────────────────┤
│                │        [터미널 / 채팅창]                    │
│ [Agent Manager]│   > npm start                              │
│ 현재 작업:     │   Server running on port 3000              │
│ ✅ 파일 생성   │                                            │
│ 🔄 코드 작성중 │   [💬 AI에게 질문하기...]                   │
│ ⏳ 테스트 대기 │                                            │
└─────────────────────────────────────────────────────────────┘
```

### 편집기 (Editor)

**코드를 직접 보고 수정하는 메인 공간**

- 파일 내용이 여기 표시됨
- 직접 타이핑해서 수정 가능
- 여러 파일을 탭으로 열 수 있음

### 파일 탐색기 (Explorer)

**프로젝트의 파일/폴더 구조를 보여주는 곳**

- 왼쪽 사이드바에 위치
- 클릭하면 파일 열림
- 우클릭으로 새 파일/폴더 생성

### 에이전트 매니저 (Agent Manager)

**AI가 현재 무슨 작업을 하는지 보여주는 곳**

- 작업 계획 표시
- 진행 상황 실시간 확인
- 완료된 작업 체크 표시

### 아티팩트 (Artifacts)

**AI 작업의 결과물을 모아두는 곳**

- 스크린샷
- 작업 계획서
- 실행 로그
- 브라우저 녹화 영상

### 브라우저 미리보기

**만든 웹페이지를 바로 확인하는 창**

- 코드 수정하면 실시간 반영
- AI가 자동으로 테스트할 때 사용

### 터미널 (Terminal)

**명령어를 입력하는 검은 화면**

- 프로그램 실행
- 패키지 설치
- 서버 시작 등

### 채팅창

**AI와 대화하는 곳**

- 한글로 질문 가능
- 코드 요청, 설명 요청, 수정 요청 등
- 여기서 Agent에게 작업 지시

## 4단계: 기본 사용법

### 프로젝트 열기

1. Antigravity 실행
2. `File` → `Open Folder` 클릭
3. 작업할 폴더 선택
4. 열기!

> 💡 새 프로젝트면 빈 폴더를 만들어서 열면 됩니다

### 파일 만들기

**방법 1: 직접 만들기**
1. 파일 탐색기에서 우클릭
2. `New File` 선택
3. 파일명 입력 (예: `index.html`)

**방법 2: AI에게 시키기**
1. 채팅창에 "index.html 파일 만들어줘"
2. AI가 자동으로 생성

### AI에게 질문하기

채팅창에 자연스럽게 한글로 질문하면 됩니다.

**예시:**
- "이 코드가 뭐하는 코드야?"
- "여기서 에러 나는데 왜 그래?"
- "버튼 클릭하면 알림 뜨게 해줘"
- "이 함수 더 간단하게 바꿔줘"

### 코드 자동완성

코드 작성 중에 AI가 다음에 올 코드를 추천해줍니다.

1. 코드 작성 시작
2. 회색 글씨로 추천 코드가 나타남
3. `Tab` 키를 누르면 적용!

### 인라인 편집 (Ctrl + K)

특정 코드만 빠르게 수정하고 싶을 때:

1. 수정할 코드 드래그해서 선택
2. `Ctrl + K` (Mac: `Cmd + K`)
3. 원하는 수정 내용 입력
4. AI가 해당 부분만 수정

**예시:**
- 코드 선택 → Ctrl+K → "이 함수에 에러 처리 추가해줘"

## 5단계: 3가지 사용 모드

Antigravity는 AI의 자율성 수준에 따라 3가지 모드를 제공합니다.

### Agent-driven 모드

**AI가 알아서 다 하는 모드**

```
나: "쇼핑몰 웹사이트 만들어줘"
AI: (혼자서 파일 생성, 코드 작성, 테스트, 수정까지 다 함)
```

- 장점: 빠름, 편함
- 단점: 뭘 하는지 파악 어려움
- 추천: 빠른 프로토타입, 간단한 작업

### Review-driven 모드

**매 작업마다 승인을 받는 모드**

```
AI: "index.html 파일을 만들까요?" → [승인] [거절]
AI: "여기에 버튼을 추가할까요?" → [승인] [거절]
```

- 장점: 모든 변경사항 확인 가능
- 단점: 매번 승인해야 해서 느림
- 추천: 중요한 프로젝트, 신중한 작업

### Agent-assisted 모드 ⭐ (추천)

**내가 주도하고, AI가 보조하는 모드**

```
나: (직접 코드 작성)
나: "이 부분 잘 모르겠어, 도와줘"
AI: (해당 부분만 도와줌)
```

- 장점: 학습에 좋음, 제어권이 나한테 있음
- 단점: 직접 해야 할 게 많음
- 추천: **초보자**, 학습 목적

> 💡 **초보자 팁**: Agent-assisted로 시작해서 익숙해지면 Agent-driven으로!

## 6단계: Agent 개념 이해 (핵심!)

이 부분이 Antigravity의 가장 중요한 특징입니다.

### Agent란?

**혼자서 일하는 AI 직원** 같은 존재입니다.

일반 직원과 비교:
```
일반 직원에게 일 시키기:
"이 보고서 작성해줘" → 직원이 작성 → 완료 보고

Agent에게 일 시키기:
"로그인 기능 만들어줘" → Agent가 작성 → 완료 보고
```

### 일반 AI 채팅 vs Agent

| 일반 AI 채팅 (ChatGPT 등) | Agent |
|--------------------------|-------|
| 질문하면 **답변**만 줌 | 질문하면 **직접 실행**함 |
| "코드는 이렇게 짜면 돼요" | (실제로 코드를 짜서 파일 저장) |
| 복사/붙여넣기 필요 | 자동으로 적용됨 |
| 한 번에 한 가지만 | 여러 작업 순차 진행 |

### Agent가 할 수 있는 것

```
┌─────────────────────────────────────────────────┐
│                  Agent의 능력                    │
├─────────────────────────────────────────────────┤
│  📝 코드 작성      - 새 코드 생성               │
│  📁 파일 관리      - 파일 생성/수정/삭제         │
│  🖥️ 터미널 사용    - 명령어 실행, 서버 시작      │
│  🌐 브라우저 제어  - 웹페이지 테스트             │
│  🔍 코드 분석      - 버그 찾기, 개선점 제안      │
│  🧪 테스트 실행    - 자동으로 테스트 수행        │
│  🔧 버그 수정      - 발견한 버그 자동 수정       │
└─────────────────────────────────────────────────┘
```

### Agent의 작업 흐름

```
1. 명령 받기
   "회원가입 페이지 만들어줘"
        ↓
2. 계획 수립
   - HTML 파일 생성
   - 입력 폼 추가
   - 유효성 검사 추가
   - CSS 스타일링
        ↓
3. 실행
   (각 단계를 순서대로 수행)
        ↓
4. 검증
   - 브라우저에서 테스트
   - 오류 있으면 자동 수정
        ↓
5. 완료 보고
   "회원가입 페이지 완성했습니다!"
```

## 7단계: Agent 활용법

### 작업 요청하기

채팅창에 자연스럽게 요청하면 됩니다.

**좋은 요청 예시:**
```
"로그인 페이지 만들어줘.
- 이메일과 비밀번호 입력 필드
- 로그인 버튼
- 비밀번호 찾기 링크 포함"
```

**안 좋은 요청 예시:**
```
"페이지 만들어줘"  ← 너무 모호함
```

> 💡 **팁**: 구체적으로 요청할수록 원하는 결과가 나옵니다!

### 계획 확인하기

Agent가 작업을 시작하면 **Agent Manager**에서 계획을 보여줍니다.

```
📋 작업 계획:
├─ ✅ 1. 프로젝트 구조 분석
├─ ✅ 2. login.html 파일 생성
├─ 🔄 3. 폼 요소 추가 (진행중)
├─ ⏳ 4. CSS 스타일 적용
└─ ⏳ 5. 유효성 검사 추가
```

마음에 안 드는 계획이 있으면 중간에 수정 요청 가능!

### 진행 상황 모니터링

실시간으로 Agent가 뭘 하는지 볼 수 있습니다:

- 어떤 파일을 수정하는지
- 어떤 명령어를 실행하는지
- 에러가 나면 어떻게 대응하는지

### 결과물 검증하기

**Artifacts 패널**에서 확인:

- 📸 스크린샷: 만들어진 화면 캡처
- 🎬 녹화: 브라우저 테스트 영상
- 📝 로그: 상세 작업 내역

### 여러 Agent 동시 실행

Antigravity의 강력한 기능!

```
Agent 1: "프론트엔드 페이지 만들어줘" (작업중...)
Agent 2: "백엔드 API 만들어줘" (작업중...)
Agent 3: "데이터베이스 설계해줘" (작업중...)
```

서로 다른 작업을 **동시에** 진행할 수 있습니다.

## 8단계: 실습 예제

직접 따라해보면서 익혀봅시다!

### 실습 1: 첫 HTML 페이지

**목표:** 간단한 웹페이지 만들기

채팅창에 입력:
```
간단한 자기소개 HTML 페이지 만들어줘.
이름, 취미, 좋아하는 음식을 포함해줘.
```

**예상 결과:**
- `index.html` 파일 생성됨
- 브라우저 미리보기에서 확인 가능

### 실습 2: 기능 추가하기

**목표:** 기존 페이지에 기능 추가

채팅창에 입력:
```
방금 만든 페이지에 "인사하기" 버튼 추가해줘.
버튼 누르면 "안녕하세요!" 알림창이 뜨게 해줘.
```

**예상 결과:**
- 버튼이 추가됨
- 클릭하면 알림창 표시

### 실습 3: 스타일 꾸미기

**목표:** 디자인 개선

채팅창에 입력:
```
페이지를 예쁘게 꾸며줘.
- 배경색: 연한 파란색
- 가운데 정렬
- 둥근 모서리 카드 스타일
```

### 실습 4: Agent 모드로 앱 만들기

**목표:** Agent가 처음부터 끝까지 만들기

채팅창에 입력:
```
간단한 계산기 웹앱 만들어줘.
- 숫자 버튼 0-9
- 더하기, 빼기, 곱하기, 나누기
- 결과 표시 화면
- 깔끔한 디자인
```

Agent Manager에서 계획과 진행 상황을 지켜보세요!

### 실습 5: 버그 수정 체험

**목표:** Agent의 디버깅 능력 체험

1. 실습 4에서 만든 계산기에서
2. 일부러 코드 일부를 지워서 에러 발생시키기
3. 채팅창에 입력:
```
계산기가 안 돼. 뭐가 문제인지 찾아서 고쳐줘.
```

Agent가 문제를 분석하고 수정하는 과정을 관찰!

## 9단계: 주의사항 & 팁

### ⚠️ 주의사항

#### AI가 완벽하지 않습니다

- 가끔 틀린 코드를 작성할 수 있음
- 항상 결과물을 검토하세요
- 이상하면 "다시 해줘" 또는 "이 부분 수정해줘"

#### 민감한 정보 주의

- 비밀번호, API 키 등을 채팅에 입력하지 마세요
- AI가 기억할 수 있습니다

#### 무료 사용량 제한

- 현재 무료이지만 사용량 제한이 있을 수 있음
- 무한정 사용 불가능할 수 있으니 참고

### 💡 꿀팁

#### 1. 되돌리기 적극 활용

- `Ctrl + Z` (Mac: `Cmd + Z`): 되돌리기
- 실수해도 걱정 마세요!

#### 2. 구체적으로 요청하기

```
❌ "버튼 만들어줘"
✅ "파란색 둥근 버튼 만들어줘.
    클릭하면 폼이 제출되게 해줘."
```

#### 3. 단계별로 요청하기

한 번에 너무 많이 요청하지 말고:

```
1단계: "기본 페이지 구조 만들어줘"
2단계: "여기에 네비게이션 추가해줘"
3단계: "로그인 폼 추가해줘"
```

#### 4. 에러 메시지 공유하기

에러가 나면:
```
"이런 에러가 나는데 해결해줘: [에러 메시지 복사]"
```

#### 5. 기존 코드 참고 요청

```
"다른 페이지랑 스타일 맞춰줘"
"기존 코드 패턴 따라서 만들어줘"
```

| 단축키 | 기능 |
|--------|------|

| `Ctrl + P` | 파일 빠른 열기 |

## 용어 정리

| 용어 | 설명 |
|------|------|
| IDE | 코드 작성 프로그램 |
| Agent | 자동으로 작업하는 AI |
| Artifacts | AI 작업 결과물 |
| 터미널 | 명령어 입력 창 |
| 디버깅 | 버그(오류) 찾아 수정하기 |
