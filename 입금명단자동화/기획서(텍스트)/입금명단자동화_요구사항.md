## 4. 네이버카페 게시 (완료)

### 4-1. 1단계 / 2단계 구조 (게시 단계 분리)
- `main.py` 실행 시 항상 **1단계(데이터 조회 → 엑셀 갱신 → 이미지 캡처 → 아카이브)**까지 수행하고,
  결과를 `run_log.json`에 `cafe_posted: false`, `post_date`(해당 회차의 날짜, YYYY-MM-DD)로 기록
- 이어서 자동으로 **2단계(`post_pending()`)**가 호출되어, `run_log.json`에서
  `image_file`이 있고 `cafe_posted: false`인 항목을 **오래된 순서대로 모두** 게시 시도
  - 제목: `{post_date} 사은품지급 명단` (post_date 없는 과거 항목은 `image_file`의 `YYYYMMDD`에서 추출)
  - 게시 성공 시 해당 항목 `cafe_posted: true`로 갱신
  - 게시 **실패 시 그 시점에서 중단**(이후 항목은 미게시로 남김) + 텔레그램 알림
- **목적**: 게시 실패 후 재시도 시 1단계(백업 선정/아카이브)가 재실행되지 않아 백업 데이터 중복이 발생하지 않음.
  휴가 등으로 며칠치 미게시가 쌓여도, 로그인만 갱신하면 한 번에 순서대로 일괄 게시 가능

### 4-2. 게시 동작
- 게시판: config.json `cafe_clubid`/`cafe_menuid` (게시판명: "사은품 당일지급 명단")
- 제목 형식: `YYYY-MM-DD 사은품지급 명단` (예: `2026-06-10 사은품지급 명단`)
- 방식: Playwright로 게시판 선택 → 제목 입력 → 이미지 업로드 → 등록
- 로그인: `chrome_profile/` 영구 프로필에 저장된 네이버 로그인 세션 재사용 (NID_AUT/NID_SES 약 30일 유효, 매 실행마다 자동 갱신됨)
  - 세션 만료/없음 → 게시 실패 + Windows 알림, `python main.py --refresh-login`으로 브라우저 수동 로그인 (해당 영구 프로필에 로그인 정보 저장)
  - 대시보드 "네이버 로그인" 카드의 경과일(25일 기준) 표시는 **프로필 내 쿠키DB 수정시각 기준**일 뿐, 실제 네이버 세션 유효 여부와는 다를 수 있음 (실제 유효성은 게시 시도 시 `_is_logged_in()`으로만 확인됨)
  - 게시 계정이 해당 게시판(메뉴ID)에 글쓰기 등급/권한이 없으면 글쓰기 페이지 접근이 막혀 `_is_logged_in()`이 False를 반환 → "로그인 세션 만료"로 오인될 수 있음. 이 경우 재로그인이 아니라 **계정 게시판 등급 확인/등업**이 필요

### 4-3. 게시 시각 (랜덤화)
- config.json `post_hour_min`/`post_hour_max` (대시보드 "실행 시각 범위"에서 변경 가능, 기본 동일값=즉시)
- Task Scheduler 트리거: 매일 `post_hour_min`:00 (대시보드에서 값 변경 시 트리거도 자동 동기화)
- 정상 스케줄 실행(`--now` 미지정, 휴일 아님)이면 `0 ~ (post_hour_max - post_hour_min)×60`분 사이 랜덤 대기 후 본 작업 시작
  → 매일 게시 시각이 범위 내에서 달라짐 (동일 시각 반복 게시로 인한 패턴 노출 완화 목적)
- 대시보드 "지금 실행"은 `--now` 플래그가 자동으로 붙어 대기 없이 즉시 실행

### 4-4. 휴일 판단
- 주말 + 공공데이터포털 공휴일 API(`holiday_api_key`) → 휴일이면 1단계 스킵 (run_log에 `skipped: '휴일/주말'`, `post_date: null` 기록)
- 휴일에도 2단계(`post_pending()`)는 호출되어, 이전에 쌓인 미게시 항목이 있으면 게시 시도

### 4-5. 봇탐지 방지 조치 (2026-06-11)
- **영구 프로필 + 실제 Chrome**: 매번 새 임시 브라우저 대신 `chrome_profile/`에 저장된 영구 프로필 + PC에 설치된 실제 Chrome(`channel='chrome'`)을 사용
  → 쿠키 외 localStorage/캐시/방문기록도 누적되어 "오래 써온 기기"처럼 동작, UA 문자열과 실제 엔진 버전(Client Hints) 불일치 문제도 해소
- **자동화 흔적 제거**: `navigator.webdriver` 오버라이드 + `playwright-stealth` 라이브러리 적용
- **사람처럼 입력/클릭**: 제목 입력은 글자마다 랜덤 딜레이, 버튼 클릭 전 마우스를 여러 지점 거쳐 이동
- **고정 대기시간 → 랜덤 범위**: 모든 대기시간을 랜덤 범위로 변경
- **탐색 동선**: 글쓰기 페이지로 직행하지 않고 카페 메인을 먼저 거쳐서 진입
- **게시 시각 랜덤화**: 4-3 참고
- **동시 실행 방지**: 영구 프로필은 동시에 두 프로세스가 열 수 없어, 스케줄러 실행 중 대시보드에서 "지금 실행"/"미게시 게시"를 누르면 자동 실패 처리됨 (중복 게시 방지 부수 효과)
- **재부팅 대비**: 작업 도중 재부팅되어도 다음 실행 시 프로필의 `Preferences`(`exit_type`)를 자동 보정하여 "페이지 복원" 알림 방지
- 최초 1회 `python main.py --refresh-login`으로 새 영구 프로필에 로그인 필요 (기존 `naver_cookies.json` 기반 세션은 이전되지 않음)

## 7. 실행 옵션 (CLI, main.py)
- `--source {wired,rental,backup,all}` — 기본 `all`(전체 파이프라인). wired/rental/backup만 단독 실행 가능
- `--capture-only` — 이미지 캡처까지만 실행, 아카이브/표시영역 초기화는 생략
- `--archive-only` — 표시 영역 데이터를 무작위 재정렬 후 아카이브만 실행
- `--refresh-login` — 네이버 로그인 (영구 프로필 `chrome_profile/`에 세션 저장, 세션 만료 시 사용)
- `--post` — 1단계 생략, `run_log.json`의 미게시(`cafe_posted: false`) 항목들을 순서대로 카페에 게시 (2단계만 실행, 대시보드 "미게시 게시" 버튼)
- `--now` — 실행 시각 랜덤 대기 생략, 즉시 실행 (대시보드 "지금 실행" 버튼)

---

## 8. 날짜 지정 (TARGET_DATE — 누락분 재처리)
- 대시보드에서 특정 날짜(예: "6/8") 지정 후 "지금실행"하면 그 날짜 기준으로 1번(B열 필터) 처리
- 처리 후 `config.json`의 `target_date`는 자동으로 `null`로 초기화됨 (1회성)
- 자동 보정(누락일 자동 감지)은 의도적으로 미구현 — 누락 시 날짜지정으로 수동 재처리하는 방식 유지
